Restaurant/Bar Application for Large Parties

Goals and Audience

To create a Restaurant/Bar App for larger parties (8+) that may want to split the bill. Parties will be ordering from the table and requires little waiter interation.

The project was intended to appeal to an average adult bar goer with some friends, who causally would like to order food or drinks with ease.

Storyboard Scenerio

Before developing this project, my team and I sketched up a possible scenerio in which a group of people would come into a bar setting and begin to use our app. Our application is linked to a barcode that can be scanned at the bar table or booth. This barcode links to our web application.

Style and Visual Design

Darker color scheme to appeal to the potential bar/late night look, easy on the eyes if in low lit places. The checkout button displays and scrolls as a header and with the content of the page. The checkout button displays how many items are in the cart at all times

Content and Layout

Checkout button placed at the top, visually it is easy to see but cannot be accidentally pressed if user is still ordering. Content is displayed into rows by category that scroll horizontally. Each row can be collapsed so that the user can streamline what they are looking for. Categories flow vertically and items within that category flow horizontally. An item can be clicked to display details and ordering options, this is to not overwhelm the user with information when initially browsing.

Navigation and Interaction

Navigation of the menu is intuitive with category names displayed in a way that is easy to see. Each item has its own picture and button. The checkout button is only displayed when there is something in the cart. The item ordering screen and the checkout screen are both separate modules that present clear footer buttons for closing or adding/ordering.
